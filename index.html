At a glance

CareLink reduces booking uncertainty for patients and speeds up scheduling work for admins using a shared availability model, clear confirmation states, and fast rescheduling tools.

Users

  • Patients booking & managing appointments
  • Clinic admins managing schedules & requests
  • Providers as part of availability constraints

What I designed

  • Patient booking: sign in → pick service → pick doctor → confirm
  • Strong confirmation + manage appointments
  • Admin: daily schedule + drawer + reschedule requests

Deliverables

  • User flow + system flow diagrams
  • Wireframes → high-fidelity UI
  • Component patterns + states

Problem

Scheduling is where patient experience and clinic operations collide. Small breakdowns create missed appointments, staff frustration, and reduced care access.

Patient pain points

  • Unclear availability (what’s actually bookable?)
  • Too many steps to confirm an appointment
  • Rescheduling feels stressful and time-consuming
  • Uncertainty after booking (did it go through?)

Admin / clinic pain points

  • Manual adjustments can cause errors & double-bookings
  • No quick way to spot conflicts in the schedule
  • Reschedules cascade across providers and rooms
  • High volume requires speed + clarity
Design challenge: One system supporting two user modes: patients (confidence + simplicity) and admins (speed + control).

Constraints & edge cases

Healthcare scheduling is full of exceptions — the UI needs to communicate states clearly.

Patient-side edge cases

  • Cancellation + rescheduling rules and cutoffs
  • Last-minute provider unavailability
  • Appointment prep notes & location clarity
  • Accessible interaction patterns (readability + tap targets)

Admin-side edge cases

  • Double-book risk (provider/room/time overlap)
  • No-show / pending / cancelled state handling
  • Reschedule requests creating cascading changes
  • High-volume updates without losing context

Outcomes & what I’d test next

If this shipped, I’d validate booking efficiency and admin speed with usability testing.

What’s strong

  • Clear patient booking path with strong confirmation state
  • Admin experience optimized for high-volume operations
  • Workflow supports common scheduling edge cases
  • State-driven UI: confirmed, pending, cancelled, no-show

Next experiments

  • Usability test: time-to-book + task success rate
  • A/B test: time-slot layout for faster selection
  • Admin test: reschedule completion time + error rate
  • Accessibility pass: contrast, focus states, keyboard flow
Portfolio takeaway: CareLink demonstrates systems-thinking UX — designing across two surfaces (patient + admin) while keeping a single source of truth for availability.
