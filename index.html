Quick summary

CareLink improves appointment booking clarity for patients and reduces scheduling overhead for admins through a shared availability model, faster rescheduling, and clear status states.

(“TL;DR” is internet shorthand meaning “Too long; didn’t read” — basically a quick summary.)

What I designed

  • Patient booking flow: choose → select → confirm
  • Manage appointments: reschedule/cancel with clear states
  • Admin schedule dashboard with quick appointment details

What I optimized for

  • Low cognitive load + high trust (healthcare context)
  • Operational speed (fewer clicks for staff)
  • Scalability (multiple providers, locations)

Key artifacts

  • User flow + system flow diagrams
  • Wireframes → high-fidelity UI
  • Component patterns + accessibility notes

Problem

Scheduling is where patient experience and clinic operations collide. Small breakdowns create missed appointments, staff frustration, and reduced care access.

Patient pain points

  • Unclear availability (what’s actually bookable?)
  • Too many steps to confirm an appointment
  • Rescheduling is stressful and time-consuming
  • Uncertainty after booking (did it work?)

Admin/clinic pain points

  • Manual adjustments cause errors & double-bookings
  • No quick way to spot conflicts in the schedule
  • Reschedules cascade across providers and rooms
  • High volume requires speed + clarity
Design challenge: build one system that supports two different user modes: patients (confidence + simplicity) and admins (speed + control).

Success metrics

Even for a concept project, I designed with measurable outcomes in mind.

↓ Steps
Reduce steps from start → confirmation

↓ Admin time
Shorten rescheduling time for staff
Target: under 60s for common reschedules
↑ Confidence
Increase clarity of availability & booking status
Target: fewer “did it book?” moments

Process

I mapped the workflows first, then designed UI that makes complex scheduling feel simple.

Solution

A unified scheduling model across patient and admin surfaces — optimized for trust, clarity, and speed.

Patient experience

  • Clear availability by provider, date, and time slot
  • Low-friction booking with a strong confirmation state
  • Manage appointments (reschedule/cancel) with clear status
  • Contextual details (location, visit type, prep notes)

Admin dashboard

  • Schedule overview for the day/week
  • Conflict awareness (double-book risk, timing overlap)
  • Fast rescheduling with minimal clicks
  • Status states: confirmed, pending, cancelled, no-show

Design decisions

Healthcare UX needs calm clarity and strong state communication.

Calm visual language

Soft contrast, generous spacing, and readable typography reduce cognitive load.

Progressive disclosure

Show essentials first, reveal details only when needed — fewer “walls of info.”

State-driven components

Clear states for booking/rescheduling prevent uncertainty and errors.

Outcomes & what I’d test next

If this shipped, I would validate flow efficiency and admin speed with usability testing.

What’s strong

  • Clear patient booking path with strong confirmation
  • Admin experience optimized for high-volume operations
  • System supports scheduling edge cases

Next experiments

  • Usability testing: booking time + error rate
  • A/B: time slot layout for faster selection
  • Admin: reschedule flow completion time
Portfolio takeaway: CareLink demonstrates systems-thinking UX — designing across two surfaces (patient + admin) while keeping a single source of truth for availability.
